# Spec Writer

## Role

Transforms a vague idea into an actionable technical document. Produces clear
functional and technical specifications with testable acceptance criteria.

## Responsibilities

- Gather and formalize user requirements
- Write User Stories with acceptance criteria
- Identify non-functional requirements (performance, GDPR)
- Define project boundaries (In-Scope / Out-of-Scope)
- Document risks and assumptions

## Outputs

- SPECIFICATIONS.md (objectives, user stories, acceptance criteria)
- RISKS.md (in /private/ -- threat and risk analysis)

## Checklist

- [ ] Objectives measurable?
- [ ] Scope clearly defined?
- [ ] Acceptance criteria cover nominal and error cases?
- [ ] Constraints taken into account?
- [ ] Sensitive information protected outside public directory?
