# POLITIQUE STRICTE : ZERO EMOJI

## PROTOCOLE D'EXÉCUTION OBLIGATOIRE
1.  **START** : Lire `docs/BACKLOG.md` et les Standards de Gouvernance.
2.  **PROCESS** : Appliquer les règles sans déviation.
3.  **END** : Mettre à jour `docs/BACKLOG.md` et `docs/CHANGELOG.md` avant tout commit.

**REGLE #0 : AUCUN EMOJI NULLE PART**

L'usage d'emojis est strictement interdit pour garantir la neutralitéé et la compatibilitéé maximale des fichiers.

- [FORBIDDEN] **Interdit** dans le CODE (y compris les commentaires).
- [FORBIDDEN] **Interdit** dans les MESSAGES DE COMMIT (git hook bloquant).
- [FORBIDDEN] **Interdit** dans la DOCUMENTATION (y compris les fichiers markdown `.md`).
- [FORBIDDEN] **Interdit** dans les fichiers de CONFIGURATION.

**Utiliser du texte explicite standardiséé à la place :**
- `[WARNING]` pour les avertissements
- `[OK]` pour les validations
- `[PRIVATE]` pour les sections confidentielles
- `[ERROR]` pour les erreurs ou interdictions
- `[INFO]` pour les notes informatives
